# 화물 혼적 경로 — 유료화·마케팅 전략 (2026-07-24)

> 결정 주제: ① 과금 모델(전체 무료체험 vs 부분 무료) ② 로그인·결제 방식 ③ 출시 준비 작업 ④ 마케팅(B2C·B2B·온라인).
> 사용자(사장님)는 현직 화물기사 — 현장 구두 홍보가 가능한 것이 최대 강점.

## 1. 핵심 결정: 과금 모델 → **2안(부분 무료) 기반 + 프리미엄만 한 달 체험** 추천

### 두 안 비교

| | 1안: 전체 무료체험 후 유료 | 2안: 기본 무료 + 프리미엄 유료 |
|---|---|---|
| 첫인상 | "어차피 유료" — 시작 장벽 | **"기본 무료"** — 부담 없음 |
| 체험 종료 시 | 앱 전체 잠김 → 삭제 위험 큼 | 기본 기능은 계속 씀 → 앱 유지 |
| 로그인 | 체험 관리 위해 초반부터 필요 | 당장 불필요 (기기 저장으로 충분) |
| 구두 홍보 | "한 달만 무료래" (약함) | **"무료니까 일단 써봐"** (강력) |
| 전환 동기 | 마감 압박 1회성 | 기록이 쌓일수록 커짐 (지속) |

### 추천안 (혼합)
- **기본 기능은 영구 무료**: 경로 최적화 · 문자 인식 · 적재 계산 · 내비 연결 · 완료보고 문자 · 주유소 탭
- **프리미엄(유료)**: 운행 기록·운송료 정산 + 정확 도착예정(실시간 교통) + (향후) 기록 백업
- **프리미엄만 첫 한 달 무료 체험** — 1안의 "맛들이기" 효과를 프리미엄에만 적용. 체험이 끝나도 앱은 계속 쓸 수 있고, 쌓인 기록은 보존되어 "내 데이터 다시 보려면 전환" 동기가 된다.
- ✅ **이미 만들어 둔 addons가 정확히 이 구조** (경로·적재·주유소 무료 / 기록·정확ETA 유료 / 31일 체험 / 기록 보존). 추가 개발 없이 문구만 "기본 무료" 강조로 다듬으면 됨.

### 왜 2안인가 (타겟 특성)
1. 화물기사층은 앱 설치·가입 절차에 매우 민감 — 가입 없이 QR 찍고 바로 써져야 첫 관문을 넘는다.
2. 이 앱의 확산 경로는 기사끼리 입소문 — "무료야, 한번 써봐"가 전달 가능한 유일한 한 문장.
3. 매일 쓰는 무료 기능이 있어야 앱이 폰에 남고, 남아 있어야 전환 기회가 온다.

## 3. 로그인·결제 — 단계적으로

### 1단계 (지금 ~ 판매 시작): 로그인 없음 + 수동 결제
- 로그인 없이 기기(폰) 저장 유지 — 마찰 최소가 최우선
- 결제: **카카오페이 송금/계좌이체 → 가입 코드 발급** (membership.js에 이미 구현됨)
  - 운영 방법: 입금 확인 → 코드 알려줌 (코드는 주기적으로 교체, ACTIVATION_CODES 수정·배포)
  - 소규모(수십 명)까지는 이 방식이 가장 빠르고 비용 0원
- ⚠️ 한계: 코드 공유 가능(지인 간). 초기엔 감수 — 오히려 확산에 도움이라고 볼 수도 있음

### 2단계 (유료 회원 ~30명 넘으면): 카카오 로그인 + 자동 결제
- **카카오 로그인**: 이미 만든 카카오 앱(ID 1521522)에서 [카카오 로그인] 활성화 + JavaScript 키 + 도메인 등록 → 웹에서 버튼 하나로 로그인. 기사들 전원 카톡 사용자라 최적.
- **서버**: Supabase(무료 티어) — 회원·구독상태·기록 백업 저장
- **정기결제**: 포트원(구 아임포트) or 토스페이먼츠 정기결제 (수수료 ~3%대). 사업자등록 필요.
- 이때 기기 저장 기록을 서버로 이전 → "폰 바꿔도 기록 유지"가 프리미엄 가치로 추가됨

### ⚠️ 유료 판매 전 행정 준비 (중요)
- **사업자등록** (간이과세 가능) + **통신판매업 신고** — 온라인 유료 서비스 판매에 필요
- 개인정보처리방침 페이지 — 로그인·서버 도입 시점부터 필수 (지금은 수집하는 개인정보 없음)

### 해석 (솔직하게)
- **손익분기는 사실상 유료 3명** — 유지비가 낮아 적자 구간이 거의 없다. 망해도 잃는 건 시간뿐.
- 시장 크기: 영업용 화물기사 약 40만+, 그중 콜·혼적 위주(1~5톤) 타겟은 5~10만 명 수준으로 추정. 1,000명 = 타겟의 1~2%로, 주선소 제휴 루프가 돌면 도달 가능한 숫자.
- 현실적 경로: 첫 3개월 지인·현장 100~300명(월 수만~수십만 원) → 1년 내 1,000명(월 35만±)이 "성공적 진행". 3,000명부터는 부업을 넘는 수익.
- 전환율이 승부처: 일반 앱 프리미엄 전환은 2~5%지만, 이 앱은 **돈(운송료 정산)이 직접 걸린 도구 + 대면 판매**라 10~20%를 노릴 수 있다. 체험 만료 시점의 안내 문구·기록 보존 정책이 그래서 중요.

| TMAP 화물차 내비 | 내비게이션 | 낮음 — 경로 안내까지가 본업, 배차문자 인식·적재·정산은 영역 밖 |

### 우리의 방어선 (카피 어려운 것 위주)
1. **중립성**: 기사들은 24시콜+원콜+지인 문자를 **섞어서** 받는다. 주선앱이 만든 도구는 자기 물량에만 작동하지만, 우리는 **어디서 온 문자든** 붙여넣으면 된다. 이 "모든 배차 소스 위의 중립 레이어" 포지션은 주선앱이 구조적으로 못 따라온다 (경쟁사 물량을 지원해줄 이유가 없으므로).
2. **데이터 락인**: 운송료 기록이 쌓일수록 이사 비용이 커진다. 지금 빨리 사용자를 모아 기록을 쌓게 하는 것 자체가 해자(垓子).
3. **니치 깊이**: 혼적 순서 최적화 + 적재함 배치 그림 + 건별 정산은 대형사가 우선순위에 올리기 애매한 "작지만 매일 쓰는" 기능. 대형사가 흉내 내도 깊이에서 이기는 게임을 한다.
4. **속도**: 1인 + AI 개발 체제는 기능 요청 → 배포가 하루. 대기업 조직은 분기 단위. 사용자 요청을 그날 반영하는 것이 최고의 차별화.
5. **현장 신뢰**: 현직 기사가 만든 앱이라는 스토리 + 대면 네트워크는 카피 불가능.

### ⚠️ 즉시 조치 필요: 코드 공개 리스크
현재 저장소가 **공개(public)** 라 후발주자가 코드를 통째로 복제할 수 있다. 유료화 전에:
- 저장소 **비공개 전환** + 호스팅을 Cloudflare Pages(무료, 비공개 저장소 지원)로 이전 권장
- 카카오 API 키도 코드에 노출 중 — 비공개 전환으로 함께 해결됨

### 만약 대형사가 정면으로 들어오면
- 그 시점은 이미 우리에게 사용자·기록·주선소 제휴가 있는 상태를 만들어 두는 것이 목표 (그래서 지금 속도가 중요)
- 최악이 아니라 기회일 수도: 니치를 검증한 팀은 **제휴·인수 대상**이 된다. "카카오T가 들어오면 진다"가 아니라 "들어오기 전에 팔 수 있는 것을 만든다"로 접근.
