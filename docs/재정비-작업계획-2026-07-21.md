# 화물 앱 재정비 작업 계획 (2026-07-21)

> 이 문서는 화물혼적경로 앱을 **① 최적 경로 + ② 적재(시각화)** 두 기둥으로 재정비하기 위한 실행 계획이다.
> 현재 코드 기준점: 커밋 `8f739bb`(글씨크기·정산 기능 도입 전 안정 상태). 이후 작업은 태그 `backup-before-revert-20260721`에 보존.

## 0. 핵심 원칙
- **두 기둥만 남긴다: 경로 / 적재.** EV·운행기록(기록)은 제거.
- **혼적**(상차지·하차지 여러 곳)이 이 앱의 존재 이유. 단일 왕복이 아니라 여러 지점을 가장 효율적으로.
- 오류 없이, 직관적으로, **고령·저시력도 한눈에**.
- 지난 실패 교훈: **화면 전체 확대(zoom) 방식 금지.** 글자 크기만 고정으로 크게. (zoom이 좁은 폭에서 글자를 한 줄에 한 단어씩 쪼갠 원인)

## 1. 확정된 결정 (2026-07-21 사용자 확인)
1. **이미지(사진) 인식 → 나중에.** 지금은 문자(텍스트) 붙여넣기만. (사진 OCR은 유료 API 필요)
2. **적재 시각화 → 2D 먼저** (위에서 본 배치도 + 옆에서 본 단수). 3D는 다음 단계.
3. **화면 = 2탭 (경로 / 적재).** + 네비에 출발지·경유지·도착지 자동 입력해 활용을 도움.
4. **글씨 = 고정 큰 글씨 하나.** 줌 토글 없음.

## 2. 화면 구조
5탭(경로·운행·적재·EV·기록) → **2탭(경로 · 적재)**.
- 운행 중 필요한 것(다음 목적지, 전화/도착문자)은 **경로 결과 화면 안에** 통합.

## 3. 경로 탭
### 3-1. 입력 자동 인식 (문자 붙여넣기 — 기존 sms.js 유지·보강)
붙여넣으면 자동 추출:
- 상차지 / 하차지 주소
- 일정 유형(당상 / 내착 등)
- 특이사항(요청·주의 문구)
- **상차 담당자·연락처 / 하차 담당자·연락처**
- 화물 정보(치수·수량) → 적재 탭으로 자동 전달

### 3-2. 최적 경로
- 여러 상차·하차 지점의 방문 순서 최적화(기존 route.js: OSRM + 자체 TSP 유지)
- 지도에 순서 표시

### 3-3. 네비 연동 (자동 입력)
- **네이버지도**: 출발지 + 경유지(최대 5) + 도착지를 URL로 한 번에 전달
  - `nmap://route/car?slat&slng&sname&dlat&dlng&dname&v1lat&v1lng&v1name...&appname=`
- **TMAP / 카카오맵**: URL은 목적지 1곳만 지원 → "도착 후 다음 목적지" 순차 버튼으로 이어감(기존 방식 유지)
- 화면에 어느 방식인지 명확히 안내

### 3-4. 원터치 연락
- 담당자별 📞 전화 / ✉️ 문자 버튼
- 정형(프리셋) 문자, 분 조절 가능:
  - "지금 **상차**하러 가는 길입니다. **5분** 후 도착합니다."
  - "지금 **하차**하러 가는 길입니다. 5분 후 도착합니다."

## 4. 적재 탭 (핵심)
### 4-1. 입력
- 화물 종류: **박스 / 파레트 / 롤** — 가로×세로×높이, 수량, 개당(또는 총) 무게
- 내 차 적재함 규격 저장(예: 1톤 저상 윙바디)
- 문자에서 넘어온 화물 자동 누적

### 4-2. 계산 (단순 CBM 아님)
- 실제 배치 기준: 방향(눕히기/세우기 회전), 단수(높이로 몇 단), **못 쓰는 빈 공간**까지 반영
- 무게: 총 적재중량 vs 최대 적재량 체크
- 혼적: 여러 화물을 앞에서부터 채워 **몇 개 실리고 몇 개 못 싣는지**, 남는 공간 크기

### 4-3. 시각화 (2D)
- **위에서 본 배치도**: 화물별 색 블록, 남는 공간 점선 표시
- **옆에서 본 모습**: 몇 단으로 쌓이는지 + 위 빈 공간(cm)
- 화물 종류·상태를 색으로 구분
- (다음 단계) 입체(3D)

## 5. 디자인 (고령·저시력 배려)
- 편안한 **고정 큰 글씨**(본문 16px 이상 기준, 줌 없음)
- 필요한 것만, 한눈에. 상차=주황 / 하차=초록 색 구분
- 큰 터치 영역(버튼·배지 44px 목표)
- Claude 디자인 원칙(artifact-design)과 협의

## 6. 단계별 실행 (각 단계 배포 후 실제 화면 확인)
- **0단계**: EV·기록 제거, 2탭 정리, 고정 큰 글씨 적용 — 안정 상태 확보
- **1단계**: 경로 입력·인식·네비 자동입력(네이버 경유지)·연락 버튼·도착 문자
- **2단계**: 적재 계산(실제 배치·무게) + 2D 시각화(위+옆) 재구축
- **3단계**: 적재 3D(입체) 시각화 — 선택
- **4단계**: 이미지(사진) 인식(OCR) — 유료 API 결정 후
