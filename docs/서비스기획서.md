# 화물 혼적 경로 — 정식 서비스 개발 기획서

> **비전**: 단순 내비게이션이 아닌 **'화물기사 전용 AI 운행 비서'** —
> 방문 순서 최적화 · 내비 연동 · 배송 관리 · 적재 계산 · EV 충전 관리를 하나로 묶은
> 국내 최고의 AI 화물 운행 플랫폼.

현재 폴더의 웹앱은 **동작하는 MVP(0단계)** 입니다. 이 문서는 이를 실제 서비스로
키우기 위한 설계입니다.

---

## 1. 전체 시스템 아키텍처

```
[모바일/웹 클라이언트 (Next.js PWA)]
        │ HTTPS
        ▼
[API 서버 (Next.js API Routes / NestJS)] ──── [Redis 캐시]
        │                                        (지오코딩·경로 캐시)
        ├─▶ [경로 최적화 엔진]  TSP 솔버 (자체) + 카카오모빌리티/TMAP 경로 API
        ├─▶ [AI 서비스]        Claude API — 운행 전략·이상 상황 조언·자연어 입력 해석
        ├─▶ [외부 API 게이트웨이] TMAP · 카카오 · 한국도로공사(통행료) · 환경부(충전소) · 기상청
        └─▶ [Supabase (PostgreSQL)]
              ├ 인증 (전화번호/카카오 로그인)
              ├ 운행·배송 데이터
              └ Realtime (관제/공유 기능)
```

- **클라이언트**: Next.js + TypeScript + Tailwind CSS, PWA(홈 화면 설치·오프라인 캐시). 운전 중 사용을 고려해 큰 버튼·음성 안내(TTS).
- **API 서버**: 외부 API 키를 서버에 숨기고, 사용량 제어·캐싱·과금 근거 기록.
- **최적화 엔진**: 지점 12개 이하는 정확해(Held-Karp), 이상은 휴리스틱(NN+2-opt, 대규모는 OR-Tools). 시간 창(도착 약속 시간) 제약이 붙으면 VRPTW로 확장.
- **AI 계층**: 경로 계산은 알고리즘이, **판단과 조언은 LLM**이 담당 — "오늘 5건 중 분당 건은 14시 약속이니 두 번째로 돌고, 퇴근 정체 전에 김포를 끝내세요" 같은 전략 브리핑.
- **캐시 전략**: 동일 주소 지오코딩 30일, 경로 결과 10분(실시간 교통 반영 주기) 캐시로 API 비용 60~80% 절감.

## 2. 데이터베이스 설계 (PostgreSQL / Supabase)

```sql
users         (id, phone, name, plan[free|pro|biz], created_at)
vehicles      (id, user_id, type, is_ev, battery_kwh, efficiency_km_kwh,
               bed_l_cm, bed_w_cm, bed_h_cm, payload_kg)
trips         (id, user_id, vehicle_id, status[planned|driving|done],
               origin_label, origin_point GEOGRAPHY,
               planned_distance_m, planned_duration_s, planned_toll_krw,
               actual_toll_krw, charge_cost_krw, charge_count,
               started_at, ended_at, memo)
trip_stops    (id, trip_id, seq, label, point GEOGRAPHY,
               work_type[load|unload], work_min,
               eta, etd, arrived_at, done_at,
               time_window_start, time_window_end)   -- 도착 약속 시간
cargo_items   (id, trip_id, name, box_w_cm, box_d_cm, box_h_cm,
               qty, unit_weight_kg, load_stop_id, unload_stop_id)
ev_logs       (id, trip_id, station_name, kwh, cost_krw, charged_at)
route_cache   (hash, request_json, response_json, expires_at)
```

- PostGIS(GEOGRAPHY)로 좌표 저장 → 반경 검색·경로 이탈 감지.
- `cargo_items`가 상차지/하차지와 연결되는 것이 핵심 — **혼적(여러 화주 짐 섞어 싣기) 관리**의 기반.

## 3. API 설계 (REST)

| 메서드 | 경로 | 설명 |
|---|---|---|
| POST | `/api/geocode` | 주소·붙여넣은 텍스트 → 좌표 (AI가 자유 형식 주소 해석) |
| POST | `/api/routes/optimize` | 출발지+배송지 목록 → 최적 방문 순서·거리·시간·통행료 |
| POST | `/api/routes/refresh` | 운행 중 실시간 교통 반영 재계산 |
| GET | `/api/tolls?path=` | 한국도로공사 API 기반 정확한 통행료 |
| POST | `/api/trips` / PATCH `/api/trips/:id` | 운행 생성·상태 변경(도착/완료 체크) |
| GET | `/api/trips?month=` | 운행 기록·월별 통계(정산용) |
| POST | `/api/cargo/calc` | 적재 계산 (3D bin packing) |
| GET | `/api/ev/stations?lat=&lng=` | 환경부 실시간 충전소(사용 가능 여부 포함) |
| POST | `/api/ai/brief` | 오늘 운행 전략 브리핑 (Claude) |
| POST | `/api/ai/parse` | "내일 아침 강남 3건, 분당 2건…" 자연어 → 배송지 목록 |

응답 예 (`/api/routes/optimize`):
```json
{
  "order": [2, 0, 1, 3],
  "total": { "distance_m": 148000, "duration_s": 9540, "toll_krw": 4200 },
  "legs": [{ "distance_m": 12300, "duration_s": 1500, "eta": "09:45" }],
  "polyline": "인코딩된 경로선",
  "ai_tips": ["출근 정체를 피해 9:30 출발 추천", "..."]
}
```

## 4. 화면(UI) 설계

**원칙: 장갑 낀 손으로도 누를 수 있게.** 핵심 흐름은 버튼 3개(현재위치 → 배송지 추가 → 최적 경로 계산).

1. **홈(경로)** — 출발지 / 배송지 붙여넣기 / [⚡ 계산] / 결과(순서·거리·시간·통행료) / 내비 3버튼 — *현재 MVP에 구현됨*
2. **운행 모드** — 전체 화면에 '다음 목적지' 하나만 크게. 도착·완료 버튼, 남은 건수, 음성 안내. 자동 도착 감지(GPS 반경 200m).
3. **적재** — 차량 선택 + 박스 입력 → 적재율 게이지, 3D 적재 배치도(정식 버전)
4. **EV** — 배터리 게이지, 경로 위 충전 필요 지점 표시, 충전소 실시간 현황
5. **기록/정산** — 월별 운행·수입·비용, 엑셀 내보내기(세금·지입 정산용)
6. **설정** — 차량 등록, 자주 가는 곳 즐겨찾기, 선호 내비 지정

## 7. 수익화 모델

**B2C (개인 기사)**
- 무료: 하루 2회 최적화, 배송지 8곳
- **프로 월 9,900원**: 무제한 최적화, AI 브리핑, 자동 도착 감지, 정산 리포트, EV 플래너
  → 기름값·통행료를 월 몇만 원만 아껴도 본전이라는 명확한 가치 제안
- 손익분기: 유료 전환 5%면 1만 MAU × 500명 × 9,900원 ≈ **월 495만원** > 운영비

**B2B (진짜 수익원)**
- 소형 물류사·지입 차주 관리 업체용 **관제 대시보드**: 차량당 월 15,000~30,000원
- 화주 → 기사 **혼적 오더 매칭** 수수료 (장기)
- 최적화 **Open API** 판매: 쇼핑몰 자체배송·식자재 유통사 대상 건당 과금

## 8. 차별화 핵심 기능 (경쟁 서비스가 못 하는 것)

1. **방문 순서 자동 최적화** — TMAP/카카오내비는 경유지 순서를 사용자가 정해야 함. 우리는 순서 자체를 AI가 계산 (핵심 차별점)
2. **혼적 관리** — 상차/하차 짝을 알고 "나중에 내릴 짐을 안쪽에" 적재 순서까지 제안. 화물 특화 서비스만 가능
3. **전기 화물차(봉고3 EV·포터 일렉트릭) 최적화** — 경로+배터리+충전소를 한 번에 계산. EV 화물차 보급 급증 중이나 전용 도구 부재
4. **적재 계산 내장** — CBM·적재율·배치까지. 견적 단계부터 사용하게 되어 자연스러운 유입 경로
5. **자연어 입력** — 화주가 보낸 카톡/문자 주문을 그대로 붙여넣으면 AI가 배송지 목록으로 변환
6. **운행 기록 = 정산 자료** — 지입/개인사업자 기사의 월 정산·세금 증빙 자동화

## 9. '매일 쓰는 앱'이 되기 위한 완성도 개선 아이디어

- **자동 도착 감지**: 목적지 200m 안에 들어오면 자동으로 도착 체크 + 다음 목적지 내비 실행 제안 — 운전 중 조작 최소화의 핵심
- **음성 우선 UX**: "다음은 분당, 14km 20분" TTS 안내. 시선을 화면에서 떼게 함
- **화주별 단골 주소록**: 두 번째 입력부터는 두 글자만 쳐도 완성
- **시간 창 제약**: "이 집은 14시까지" 입력 시 순서 최적화에 반영 (VRPTW)
- **경로 이탈·지연 감지**: 예정보다 30분 늦으면 남은 일정 자동 재계산 + 화주 공유용 링크
- **오프라인 대비**: 산간·지하 주차장에서도 마지막 경로·체크리스트는 항상 표시
- **커뮤니티 데이터**: 기사들이 남기는 "이 집 하차 30분 걸림" 메모 → 다음 기사 ETA에 반영 (네트워크 효과·해자)
- **차종 제한 반영**: 높이·중량 제한 도로 회피 (TMAP 화물차 경로 API)
- **다크모드·야간 운행 모드** 기본 (현재 MVP도 다크 기반)
